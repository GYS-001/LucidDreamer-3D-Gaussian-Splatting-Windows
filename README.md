# LucidDreamer 3D Gaussian Splatting  Windows 全流程工程化落地（无 Point-E 纯本地训练）
## 项目概述
本项目基于 **LucidDreamer** 文本驱动3D高斯溅射生成框架，**在 Windows 非原生支持平台**完成从底层CUDA算子编译、核心模块重构、训练链路打通到模型收敛的**全流程工程化落地**。
核心目标：**彻底移除项目对 Point-E 预训练模型的外网依赖**，实现**断网纯本地、零外部模型、稳定可复现**的3D生成训练，解决开源项目国内落地的网络与环境痛点。

## 技术栈
- 编程语言：Python、C++、CUDA C++
- 深度学习框架：PyTorch
- 核心技术：3D Gaussian Splatting、Stable Diffusion 文本引导、点云初始化、多视角渲染
- 运行平台：Windows（原生非支持平台）
- 核心特性：**无 Point-E、无外网依赖、纯本地初始化、全流程稳定训练**

## 核心工程改造（项目核心）
本项目并非简单运行开源代码，而是完成**底层级工程改造与问题攻坚**，核心改造如下：
1. **Windows 平台 CUDA 扩展算子本地编译**
   3D高斯官方仅支持Linux，本项目独立完成**光栅化、3D协方差计算、球谐投影**等核心CUDA算子的源码编译，解决VS编译器兼容、CUDA版本联动、PyTorch ABI冲突等底层工程问题。
2. **移除 Point-E 外网依赖，自研纯本地点云初始化**
   剔除项目对 HuggingFace/Point-E 预训练模型的强网络依赖，**自研梯度增强立方体点云初始化策略**，替代原预训练点云方案，实现断网环境一键启动训练。
3. **底层代码 Bug 修复与训练稳定性优化**
   修复 PyTorch 张量类型不兼容、整数张量求导异常、高斯参数维度不匹配、梯度累积溢出等隐性底层Bug，实现千轮以上无崩溃、无报错的稳定训练。
4. **3D生成训练调优与行业共性问题解决**
   针对纯随机初始化下**模式坍缩（球化/噪声团）、轮廓退化、语义跑偏、致密化失效**等问题，完成分阶段训练策略、超参数调优、Prompt正负向约束优化，保证模型稳定收敛。

## 环境与依赖
- 系统：Windows 10/11
- CUDA：11.8
- PyTorch：2.0+ 对应CUDA版本
- 编译器：Visual Studio 2022（MSVC）
- 其他依赖：requirements.txt 标准Python依赖库

## 全流程执行步骤
### 1. 底层环境编译
1. 配置 Windows CUDA 编译环境，联动 VS2022 与 CUDA Toolkit
2. 本地编译 3D 高斯全套 CUDA 扩展算子，验证算子无编译报错
3. 安装 Python 依赖，验证 PyTorch-CUDA 环境可用

### 2. 核心模块替换
1. 注释原项目 Point-E 初始化相关代码
2. 替换为**梯度增强立方体纯本地点云初始化模块**
3. 验证初始化无报错，初始点云梯度正常可驱动训练

### 3. 训练配置调整
优化 yaml 核心参数，适配纯本地初始化训练：
- 延长几何训练阶段，固定形状后再优化纹理
- 降低致密化触发阈值，抑制模式坍缩
- 强化文本引导与负向约束，解决语义跑偏
- 限制最大迭代步数，避免轮廓过度磨平

### 4. 启动训练
```bash
python train.py --opt run_example.yaml
```
训练全程自动保存：迭代视图、高斯模型、训练日志，支持断点追溯。

## 核心踩坑与解决方案
| 问题场景 | 问题描述 | 解决方案 |
| ---- | ---- | ---- |
| Windows编译报错 | CUDA算子与MSVC编译器不兼容、模板编译失败 | 适配CUDA版本、修正编译参数、解决ABI冲突 |
| 外网依赖阻断 | Point-E模型国内无法下载，项目无法启动 | 移除预训练依赖，自研纯本地点云初始化 |
| 训练模式坍缩 | 模型收敛为圆球/噪声团，无有效结构 | 优化初始化梯度、调整分阶段训练策略 |
| 语义跑偏问题 | 文本引导失效，生成无关物体（人头） | 优化3D专属Prompt、添加强负向约束 |
| 致密化失效 | 高斯点数不增长，细节无法细化 | 降低梯度阈值、提升初始点云梯度强度 |
